# Pomlist 产品简报

> 当前仓库已刻意清空旧实现，只保留这份产品定位与功能概述，作为后续重写 Pomlist 的参考资料。

## 项目定位

Pomlist 是一个“任务驱动”的番茄钟与个人执行管理工具。它不把专注简单理解为固定分钟数倒计时，而是把一次专注定义为：围绕一组明确任务，在一段时间内推进到可复盘的状态。

核心目标是让用户在开始前选清楚要推进什么，在进行中持续看见完成进度，在结束后得到可回看的任务、时长、完成率和效率数据。

## 核心理念

传统番茄钟关注“时间是否结束”，Pomlist 更关注“任务是否被推进”。一次 Pomlist 专注会话应该同时回答三个问题：

- 本轮开始时，我选择了哪些任务？
- 本轮进行中，我完成了哪些任务？
- 本轮结束后，我获得了什么样的产出与效率反馈？

因此，Pomlist 的产品重心不是单纯计时，而是任务选择、专注执行、过程勾选、结束复盘和长期统计。

## 目标用户

- 希望用番茄钟约束工作节奏，但又不想把任务管理和计时工具分开的个人用户。
- 需要记录每次专注产出、回看历史表现、观察效率变化的学习者或独立开发者。
- 喜欢轻量、本地优先、启动即用体验的 iPhone / iPad 用户。

## 功能范围

### 身份与本地使用

- 4 位口令登录。
- 支持修改口令与退出登录。
- 默认面向单用户、本机使用。
- 数据优先保存在本地，未来可再扩展同步能力。

### 任务库

- 新增、编辑、删除任务。
- 任务支持标题、备注、分类、标签、状态。
- 任务状态至少包含待办、已完成、已归档。
- 分类和标签应能复用，避免重复输入。

### 专注会话

- 用户从任务库中选择一个或多个任务开始本轮专注。
- 同一时间只允许存在一个进行中的专注会话。
- 会话进行中可以继续追加任务。
- 会话中可以逐项勾选任务完成状态。
- 会话结束时记录开始时间、结束时间、持续时长、任务快照、完成数量和完成率。

### 今日视图

- 展示当前进行中的专注会话。
- 展示今日可推进任务与核心进度。
- 提供开始专注、结束专注、追加任务、勾选任务等主操作。

### 任务视图

- 管理完整任务库。
- 支持按状态、分类或关键词筛选。
- 支持快速完成、恢复、删除任务。
- 支持维护分类和标签。

### 历史视图

- 按时间倒序展示已结束的专注会话。
- 每条历史记录应展示会话时长、任务数量、完成数量、完成率和任务快照。
- 历史记录用于复盘，不应依赖当前任务是否被修改或删除。

### 统计视图

- 展示今日、近 7 天、近 30 天等周期指标。
- 统计专注次数、总时长、完成任务数、平均完成率。
- 展示连续专注天数。
- 展示分类贡献、时段分布和效率趋势。

### 设置

- 修改口令。
- 退出登录。
- 后续可扩展数据导出、数据清空、同步设置、主题设置等能力。

## 信息结构

```text
Pomlist
├── Today：当前专注、今日任务、进度操作
├── Tasks：任务库、分类、标签、任务状态
├── History：已结束专注会话与任务快照
├── Stats：周期统计、分类贡献、效率趋势
└── Settings：口令、安全、本地数据与偏好
```

这套结构的重点是把“今天要做什么”和“长期任务管理”分开，同时让历史和统计承担复盘职责。

## 数据模型轮廓

### Task

- `id`：任务唯一标识。
- `title`：任务标题。
- `notes`：任务备注。
- `category`：任务分类。
- `tags`：任务标签。
- `status`：任务状态。
- `createdAt`：创建时间。
- `updatedAt`：更新时间。
- `completedAt`：完成时间。

### FocusSession

- `id`：会话唯一标识。
- `state`：进行中或已结束。
- `startedAt`：开始时间。
- `endedAt`：结束时间。
- `elapsedSeconds`：持续秒数。
- `tasks`：本轮任务快照。
- `totalTaskCount`：任务总数。
- `completedTaskCount`：完成任务数。
- `completionRate`：完成率。

### SessionTaskSnapshot

- `id`：快照唯一标识。
- `taskId`：原任务标识。
- `titleSnapshot`：开始或加入会话时的任务标题。
- `categorySnapshot`：当时的分类。
- `tagSnapshot`：当时的标签。
- `orderIndex`：本轮排序。
- `isCompletedInSession`：是否在本轮完成。
- `completedAt`：本轮完成时间。

## 产品原则

- 专注围绕任务，而不是围绕倒计时本身。
- 会话记录必须可复盘，不能因为任务后续编辑而丢失历史语义。
- 常用操作应该足够快：开始、勾选、结束、追加任务都应少步骤完成。
- 本地优先，离线可用，数据结构清晰。
- UI 文案保持克制，控件通过标签、状态、布局和图标表达含义。
- 统计服务于行动，不堆叠低价值图表。

## 后续重写边界

后续重写可以自由选择技术栈与实现方式，但应保留以下产品内核：

- 任务库与专注会话是两个独立但相互关联的概念。
- 专注会话保存任务快照。
- 同一时间只允许一个 active session。
- 历史记录和统计基于已结束会话生成。
- 首版优先保证本地数据可靠，再考虑云同步和多端协作。
